Why should I trust you with my login data?

Good question, thanks for asking.
You shouldn't. It's as simple as that. Trust no-one who wants your login data for anything, other than the original company you've made that info for.

This is also why you can login using the token, as that one is easily invalidated and doesn't allow anyone to steal your account details.
And I would really like for this application to be server-less, browser/client side only, but sadly mojangs api servers don't allow requests from browsers directly, so I need a server to relay that request for you to be able to access the api. I've made the code open source, in part for that reason, so that everyone can open it up and see that nothing is logged other than error messages, that nothing is saved on the server and everything is saved in your browser (oh yeah btw, I'm using cookies for that).
